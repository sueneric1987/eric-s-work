        <property name="beanNames">

11、spring配置文件中，parent属性用作对父类属性的重新配置
<bean id="parentBean" class="com.eric.bean.ParentBean">
  <property name="prop" value="father">
</bean>
<bean id="childBean" class="com.eric.bean.ChildBean" parent="parentBean">
  <property name="prop" value="child">
</bean>

此时容器启动时，childBean的实例的prop是child，如果没有配置属性，prop是father

spring事务总结

使用声明式事务管理
@Transactional(propagation=Propagation.REQUIRED) 
public void a(){
        testa//事务方法
	try{
	   b(){
               testb//事务方法
               
	   };    
           b方法现在没有使用代理，也就是没有事务切面控制，那么如果b中抛出的错不是数据库错误并且在b事务方法前，那么a中事务提交，b中事务失败
                                                              如果b中抛出的错不是数据库错误并且在b事务方法后，那么a中事务提交，b中事务成功
                                                              如果b中抛出的错是数据库错误，那么a中事务失败，b中事务也失败
           b方法现在使用代理，有事务切面控制，那么如果b中只要抛错，a和b的事务都失败，报错是Transaction rolled back because it has been marked as rollback-only
                                                              
	}catch(exception e){
	   //
	}
}
